# Eval Comparison Notes

Use this file to track how TON-auditor changes affect benchmark behavior.

For each benchmark, record:

- benchmark name
- audit mode
- expected high-confidence findings
- expected below-threshold findings, if any
- false positives that must not be reported
- source language and expected language-specific agent coverage
- notes on missed parser-integrity, receiver/fallback, typed-decoder, standards, bounce, gas, or message-flow issues

Prefer concrete, objectively checkable expectations over broad quality judgments.
